# i18n Español + Portugués — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Spanish/Portuguese i18n to the EggPertise Next.js 14 App Router site using `next-intl` for UI strings and Contentful native locales for dynamic content.

**Architecture:** All app routes move under `app/[locale]/`. A `middleware.ts` auto-detects browser language and redirects `/` to `/es` or `/pt`. `next-intl` provides typed translations from `messages/es.json` and `messages/pt.json`. Contentful queries receive a locale param mapped from the URL locale (`es→en-US`, `pt→pt-BR`).

**Tech Stack:** next-intl 3.x, Next.js 14 App Router, Contentful GraphQL, Contentful MCP

- [ ] **Step 3: Create `i18n/request.ts`**

```ts
import { getRequestConfig } from "next-intl/server";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (!locale || !routing.locales.includes(locale as "es" | "pt")) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

const CONTENTFUL_LOCALE_MAP: Record<string, string> = {
